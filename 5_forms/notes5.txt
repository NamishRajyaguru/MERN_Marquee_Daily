========================================================================
ACADEMIC NOTES: HTML FORMS AND USER INPUT
========================================================================

1. THE <form> ELEMENT
--------------------------------------------------
- Purpose: The `<form>` element acts as a container for interactive controls used to collect information from a user and submit it to a server.
- Key Attributes:
  - `action`: Specifies the URI (URL) of the program or script on the server that will process the submitted form data.
  - `method`: Specifies the HTTP method used to send the data.
    - `GET`: Appends form data to the URL in name/value pairs. Used for retrieving data; should never be used for sensitive information (passwords). It is idempotent.
    - `POST`: Sends form data inside the body of the HTTP request. Used for sending sensitive data or large amounts of data. It is non-idempotent (submitting twice creates two transactions).
  - `enctype`: Specifies how the form data should be encoded (crucial for file uploads, which require `multipart/form-data`).

2. THE <input> ELEMENT AND TYPES
--------------------------------------------------
The `<input>` element is the most versatile form control, defined primarily by its `type` attribute.
- Textual Inputs: `type="text"`, `type="password"`, `type="email"`, `type="url"`, `type="tel"`, `type="search"`. HTML5 types provide automatic client-side validation and trigger appropriate virtual keyboards on mobile devices.
- Numeric/Quantitative: `type="number"` (spinner), `type="range"` (slider).
- Choice Inputs:
  - `type="radio"`: Mutually exclusive selection. Radio buttons that share the same `name` attribute form a group where only one can be selected.
  - `type="checkbox"`: Independent Boolean selection. Multiple checkboxes can be selected.
- Action Inputs: `type="submit"`, `type="reset"`, `type="button"`.
- Specialized: `type="date"`, `type="time"`, `type="color"`, `type="file"`, `type="hidden"`.

3. OTHER FORM CONTROLS
--------------------------------------------------
- `<textarea>`: A multi-line text input control. Attributes include `rows` and `cols` to define visible dimensions.
- `<select>` and `<option>`: Creates a dropdown list. `<select>` is the container, and `<option>` defines available choices. The `multiple` attribute on `<select>` allows selecting more than one option.
- `<button>`: A clickable button. More versatile than `<input type="button">` as it can contain nested HTML content (like images or icons).

4. ACCESSIBILITY AND USABILITY (<label> & <fieldset>)
--------------------------------------------------
- `<label>`: Defines a label for a form element.
  - Mechanism: The `for` attribute of the `<label>` MUST match the `id` attribute of the corresponding `<input>`.
  - Benefits: Dramatically improves accessibility (screen readers announce the label when the input is focused) and usability (clicking the label text activates the associated input, expanding the click target area for checkboxes and radios).
- `<fieldset>` and `<legend>`: `<fieldset>` groups related elements within a form, often drawing a box around them. `<legend>` provides a caption for the `<fieldset>`.

5. FORM ATTRIBUTES AND VALIDATION
--------------------------------------------------
- `name`: Crucial for server submission. It acts as the variable name for the input's data.
- `value`: Specifies the initial or default value of the input.
- `placeholder`: Provides a short hint that describes the expected value. (Do NOT use as a replacement for `<label>`).
- Client-Side Validation Attributes:
  - `required`: Specifies that an input field must be filled out before submitting.
  - `min`, `max`: Specifies minimum and maximum values (for numbers and dates).
  - `pattern`: Specifies a regular expression that the input's value is checked against.
  - `disabled`: Disables the input (data is NOT sent upon submission).
  - `readonly`: Makes the input non-editable (but data IS sent upon submission).
